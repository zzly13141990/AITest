## 3. 业务流程梳理

### 3.1 主题切换流程
```
用户进入应用
    ↓
读取 localStorage 中的主题偏好
    ↓
    ├─ 有偏好 → 使用偏好主题
    │
    └─ 无偏好 → 检测系统主题 → 使用系统主题
    ↓
用户点击主题切换按钮
    ↓
应用新主题样式
    ↓
保存新主题到 localStorage
```

### 3.2 系统主题跟随流程
```
用户选择"跟随系统"模式
    ↓
应用检测系统当前主题
    ↓
应用相应主题
    ↓
监听系统主题变化
    ↓
系统主题变化时自动切换应用主题
```

## 8. 潜在风险与 Bug 预判

### 8.1 已识别问题

根据代码审查，发现以下潜在问题：

| 问题描述 | 严重程度 | 修复建议 |
|---------|---------|---------|
| SqlEditorPage 中标签页区域背景硬编码为 #f5f5f5，未使用 CSS 变量 | 中 | 将硬编码颜色替换为 CSS 变量 `var(--bg-tertiary)` 等 |
| SqlEditorPage 中标签页选中状态背景硬编码为 white | 中 | 替换为 `var(--bg-card)` |
| SqlEditorPage 中标签页边框和拖拽手柄部分颜色硬编码 | 中 | 全部替换为 CSS 变量 |
| Layout 中连接项细节文字颜色硬编码为 #666 和 #999 | 中 | 替换为 `var(--text-secondary)` 和 `var(--text-tertiary)` |

### 8.2 潜在风险

| 风险描述 | 可能性 | 影响 | 建议措施 |
|---------|-------|-----|---------|
| 新增组件未使用 CSS 变量，深色模式样式缺失 | 中 | 高 | 建立代码审查机制，确保新组件使用 CSS 变量 |
| 第三方组件库深色模式适配不完整 | 低 | 中 | 选择支持主题切换的组件库，或为第三方组件写自定义样式 |
| 对比度不足，影响可访问性 | 低 | 高 | 使用 WCAG 2.1 标准检查工具定期检测 |
| 主题切换时出现闪烁 | 低 | 中 | 优化 CSS transition，或在加载时先应用主题再渲染 |

# 深色模式 UI 设计规范

## 1. 项目设计风格定位

### 1.1 设计理念
- **极简专业**: 遵循 B 端产品极简设计原则，去除不必要的装饰元素
- **低饱和度**: 采用低饱和度颜色，减少视觉疲劳
- **高对比度**: 确保文本与背景有足够的对比度，符合 WCAG AA 标准
- **品牌延续**: 保持原有品牌色绿色（#4CAF50）的一致性

### 1.2 适用场景
- 夜间工作环境
- 长时间使用
- 深色主题偏好用户
- 与操作系统主题同步

## 5. 全局导航和菜单结构设计

## 8. 交互状态设计规范

### 8.1 过渡动画
- 主题切换过渡：300ms，缓动函数 `ease-in-out`
- 按钮悬浮过渡：150ms，缓动函数 `ease-out`
- 输入框 focus 过渡：200ms，缓动函数 `ease-out`

### 8.2 加载状态
- 加载动画：旋转圆圈，颜色 `#4CAF50`
- 骨架屏：背景 `#252525`，动画渐变 `#1E1E1E` ↔ `#2D2D2D`
- 按钮加载：文字隐藏，显示加载动画

### 8.3 禁用状态
- 透明度降低：文字 `#616161`，背景色调整为深色
- 光标：`not-allowed`
- 不可点击

### 8.4 空状态
- 图标：大小 64px，颜色 `#616161`
- 文字：14px，颜色 `#9E9E9E`
- 间距：图标与文字间距 16px

### 8.5 选中状态
- 背景高亮：`#1B5E20`（绿色半透明背景）
- 边框高亮：`#4CAF50`
- 文字高亮：`#4CAF50`

## 9. 前端开发实现建议

### 9.1 CSS 变量定义
建议使用 CSS 自定义属性实现主题切换：

```css
:root {
  /* 浅色模式变量 */
  --bg-primary: #f5f5f5;
  --bg-secondary: #ffffff;
  --text-primary: #333333;
  --text-secondary: #666666;
  --border-color: #dddddd;
  /* ... 更多变量 */
}

[data-theme="dark"] {
  /* 深色模式变量 */
  --bg-primary: #121212;
  --bg-secondary: #1E1E1E;
  --text-primary: #E0E0E0;
  --text-secondary: #9E9E9E;
  --border-color: #333333;
  /* ... 更多变量 */
}

* {
  transition: background-color 0.3s ease-in-out, 
              color 0.3s ease-in-out, 
              border-color 0.3s ease-in-out;
}
```

### 9.2 Monaco 编辑器配置
```javascript
// 深色模式配置
const editorOptions = {
  theme: 'vs-dark',
  // 其他配置...
};
```

### 9.3 主题切换实现
- 使用 localStorage 存储用户偏好：`db-query-theme`
- 支持三种模式：`light`（浅色）、`dark`（深色）、`system`（跟随系统）
- 监听系统主题变化：`prefers-color-scheme` 媒体查询

### 9.4 响应式适配
- 保持现有响应式规则不变
- 确保在深色模式下各断点显示正常

## 10. 可访问性规范

### 10.1 对比度要求
- 正文文本与背景对比度 ≥ 4.5:1（WCAG AA）
- 大文本（≥18px 或 ≥14px 加粗）对比度 ≥ 3:1
- 交互组件状态有明确区分

### 10.2 焦点可见性
- Focus 状态有明确的视觉反馈
- 输入框 Focus 时显示边框高亮 + 外发光
- 按钮 Focus 时有轮廓或阴影

# 软件需求与系统设计文档 - 深色模式支持

## 1 项目概述

### 1.1 项目背景

db-query-java 是一个数据库查询工具，目前采用浅色模式设计。随着用户在低光环境下使用需求的增加，需要为项目添加深色模式支持，以提升用户体验，减少眼睛疲劳。

### 1.2 建设目标

- 为 db-query-java 项目添加完整的深色模式支持
- 提供深色/浅色模式切换功能
- 确保主题切换时的平滑过渡效果
- 保持现有功能完整性不受影响

### 1.3 适用范围

本需求适用于 db-query-java 项目的前端界面，包括：

- 主界面布局
- 数据库连接管理页面
- SQL 编辑器页面
- 查询结果展示
- 元数据导航侧边栏

### 1.4 读者对象

- 前端开发人员
- UI/UX 设计师
- 测试工程师
- 产品经理

## 2 业务需求概述

### 2.1 业务场景

#### 2.1.1 日常开发场景

数据库开发人员在夜间或低光环境下长时间使用工具进行 SQL 编写和查询，深色模式可以有效减少眼睛疲劳。

#### 2.1.2 偏好选择场景

部分用户天生偏好深色界面，提供模式切换可以满足不同用户的个性化需求。

#### 2.1.3 系统主题同步场景

用户希望应用能与操作系统的主题设置（深色/浅色）保持同步。

### 2.2 用户角色与职责

| 角色    | 职责                        |
| ----- | ------------------------- |
| 终端用户  | 使用深色/浅色模式切换功能，选择符合自己偏好的主题 |
| 前端开发  | 实现深色模式的样式、切换功能和兼容性保障      |
| 测试工程师 | 验证深色模式在各页面的显示效果和功能正常性     |

### 2.3 业务核心痛点

1. **视觉疲劳**：现有浅色模式在低光环境下使用易造成眼睛疲劳
2. **缺乏个性化**：无法满足用户对界面主题的个性化偏好
3. **系统同步缺失**：无法与操作系统的主题设置保持一致

### 2.4 整体业务流程

```
用户访问应用
    ↓
检查用户偏好/系统主题
    ↓
应用对应主题
    ↓
用户可随时切换主题
    ↓
主题偏好持久化存储
```

## 3 功能需求

### 3.1 整体功能模块架构

```
深色模式系统
├── 主题状态管理
│   ├── 状态存储
│   └── 状态读取
├── 主题切换功能
│   ├── 切换按钮
│   └── 切换动画
├── 深色主题样式
│   ├── 基础样式
│   ├── 组件样式
│   └── Monaco编辑器主题
└── 兼容性保障
    ├── 系统主题检测
    └── 持久化存储
```

### 3.2 各模块详细功能清单

| 模块     | 功能点                       | 优先级 |
| ------ | ------------------------- | --- |
| 主题状态管理 | 主题状态存储（light/dark/system） | 高   |
| 主题状态管理 | 从 localStorage 读取用户偏好     | 高   |
| 主题切换功能 | 提供主题切换按钮/开关               | 高   |
| 主题切换功能 | 三种模式可选：浅色、深色、跟随系统         | 中   |
| 深色主题样式 | 实现深色模式下的基础样式（背景、文字）       | 高   |
| 深色主题样式 | 实现深色模式下的卡片、表单、按钮样式        | 高   |
| 深色主题样式 | 实现深色模式下的表格样式              | 高   |
| 深色主题样式 | 配置 Monaco 编辑器的深色主题        | 高   |
| 兼容性保障  | 检测操作系统主题设置                | 中   |
| 兼容性保障  | 监听系统主题变化并自动切换             | 中   |

### 3.3 单个功能详细描述

#### 3.3.1 主题状态管理

**功能描述**：管理当前应用的主题状态，支持存储和读取用户的主题偏好。

**输入**：

- 用户选择的主题模式（light/dark/system）

**输出**：

- 应用对应主题的界面样式

**业务规则**：

- 首次使用时默认跟随系统主题
- 用户选择后优先使用用户偏好
- 主题状态持久化到 localStorage

#### 3.3.2 主题切换功能

**功能描述**：提供用户界面元素，允许用户在浅色、深色、跟随系统三种模式之间切换。

**输入**：

- 用户点击切换按钮/开关

**输出**：

- 界面主题立即切换
- 新的主题偏好保存

**业务规则**：

- 切换按钮应放置在明显位置（建议右上角）
- 切换时应有平滑的过渡动画
- 按钮应清晰标识当前主题状态

#### 3.3.3 深色主题样式

**功能描述**：实现深色模式下的完整样式体系。

**输入**：

- 应用切换到深色模式

**输出**：

- 所有界面元素显示深色主题样式

**业务规则**：

- 深色背景色建议使用 #1e1e1e 或 #121212
- 文字颜色使用浅色，确保足够对比度（WCAG AA 标准）
- 主色调保持品牌一致性（当前绿色 #4CAF50）
- 边框、分割线使用较暗的颜色
- 悬停、激活状态有适当的视觉反馈

#### 3.3.4 Monaco 编辑器深色主题

**功能描述**：为代码编辑器配置合适的深色主题。

**输入**：

- 应用切换到深色模式

**输出**：

- Monaco 编辑器显示深色主题

**业务规则**：

- 使用 Monaco 内置的 'vs-dark' 主题
- 或自定义深色主题与整体风格保持一致

### 3.4 页面/交互简要说明

#### 3.4.1 主题切换按钮

- 位置：右上角导航栏
- 样式：图标按钮（太阳/月亮图标）
- 交互：点击切换或下拉选择

#### 3.4.2 过渡动画

- 主题切换时应用 CSS transition
- 建议时长：300ms
- 缓动函数：ease-in-out

## 4 非功能需求

### 4.1 性能需求

- 主题切换响应时间 < 100ms
- 不影响页面加载速度
- 不增加明显的内存占用

### 4.2 安全需求

- 主题偏好存储在客户端 localStorage，无安全风险
- 不涉及敏感数据处理

### 4.3 兼容性需求

- 浏览器支持：Chrome、Firefox、Safari、Edge（最新两个版本）
- 支持操作系统主题检测（Windows、macOS、Linux）
- 响应式设计在深色模式下保持正常

### 4.4 可扩展与可维护性

- 使用 CSS 变量实现主题，便于维护和扩展
- 主题相关代码集中管理
- 便于未来添加更多主题（如护眼模式、高对比度模式等）

### 4.5 部署与运维需求

- 前端构建后自动包含深色模式样式
- 无需后端改动
- 不影响现有功能的部署和运行

## 5 数据与数据库设计

### 5.1 核心数据实体

本功能不涉及后端数据库，仅在客户端存储主题偏好。

### 5.2 存储设计

**存储位置**：localStorage

**存储键**：`db-query-theme`

**存储值**：

- `'light'` - 浅色模式
- `'dark'` - 深色模式
- `'system'` - 跟随系统

### 5.3 数据流转关系

```
用户访问应用
    ↓
检查 localStorage 中的主题偏好
    ↓
    ├─ 有偏好 → 使用偏好主题
    │
    └─ 无偏好 → 检测系统主题
                ↓
            使用系统主题
                ↓
            保存到 localStorage
```

## 6 接口设计

本功能不涉及后端接口变更，无需新增接口。

## 7 约束与边界说明

### 7.1 技术约束

- 使用 CSS 自定义属性（CSS Variables）实现主题切换
- 使用 React Context 或状态管理库管理主题状态
- 不引入新的第三方依赖库

### 7.2 设计约束

- 保持现有浅色模式不变
- 深色模式与浅色模式功能完全一致
- 品牌色（绿色 #4CAF50）保持不变

### 7.3 兼容性边界

- 对于不支持 `prefers-color-scheme` 媒体查询的浏览器，默认使用浅色模式
- localStorage 不可用时，使用会话存储或默认浅色模式

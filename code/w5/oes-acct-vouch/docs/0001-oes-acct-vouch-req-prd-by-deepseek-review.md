## 一、🔴 Critical 问题

### C-1: §8.5 特殊业务字段映射汇总表遗漏回单号映射行

**位置**: §8.5 第 1664-1669 行

**现状**：
```
| **日期** | info_fzhs{N} | order_date, occur_date |
| **结算方式** | info_fzhs{N} | pay_type_id |
| **票据号** | info_fzhs{N} | cheq_no, order_no |
| **其他自定义值** | info_fzhs{N} | 无 |
```

**问题**：
- §5.4.3、§8.2 已明确定义回单号→receipt_no 映射，但 §8.5 汇总表缺失该行
- "其他自定义值"概括了所有非特殊映射的值，但回单号是已有明确映射的特殊值，不应收归为"其他"

**影响**：
- 阅读 §8.5 的开发人员看不到"回单号"这一特殊映射，可能误认为回单号仅写入 info_fzhs
- 与 §5.4.3 和 §8.2 产生矛盾

**修正建议**：
在 §8.5 表中补充"回单号"行，放在"票据号"之后：
```
| **回单号** | info_fzhs{N} | receipt_no | 等于用户输入的回单号 | 用户输入回单号 |
```

**审查决议**: 🔴 必须修正

## 二、🟢 Suggestion 建议

### S-1: §2 术语表补充"票据号"和区分说明

**位置**: §2 术语与缩写（第 42-60 行）

**现状**：
术语表中已有"回单号(receipt_no)"，但缺少"票据号"的术语定义，也缺少两者区分的说明。

**建议**：
补充两条术语：
> | **票据号 (cheq_no / order_no)** | acct_check_items 表中的业务字段，存储"票据号"时的值，同时写入 cheq_no 和 order_no |
> | **票据号 vs 回单号** | 两者为独立概念：票据号→cheq_no+order_no；回单号→receipt_no |

**审查决议**: 🟢 建议补充

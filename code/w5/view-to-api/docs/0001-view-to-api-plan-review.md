## [W] Warning（建议修复）

### W1. 引用的设计文档版本已过时

**文件位置**: 第 1 节（概述 - 关联文档）  
**严重性**: [W] Warning  
**问题描述**:

第 1 节中关联文档引用的是 `docs/0001-view-to-api-design.md (v1.2)`，但当前最新的设计方案版本是 v1.4（增加了前端暗色设计主题 v1.3 和前后端一体化 v1.4）。

版本 v1.3 中新增了极为详细的前端暗色视觉体系（CSS 令牌、毛玻璃效果、灯笼式辉光、动效系统），v1.4 又更新了前后端一体化部署方案。这些变化直接影响了前端 Phase 3 的实现范围和部署 Phase 4 的构建流程。

**建议修复**:

将关联文档中的 Design 引用更新为 `docs/0001-view-to-api-design.md (v1.4)`，并审核计划中前端实现和部署任务的描述是否与设计 v1.4 一致。

---

### W2. 前端工期估算可能偏低（暗色主题未体现）

**文件位置**: 第 5 节（Phase 3 — 前端页面实现）  
**严重性**: [W] Warning  
**问题描述**:

当前的 Phase 3 总工时估算是 1.5 天。但设计 v1.4 中包含了极其详尽的前端暗色主题设计：
- 30+ CSS 设计令牌变量
- 毛玻璃（backdrop-filter）侧栏和筛选卡片
- 表格左侧渐变光晕装饰线
- 状态标签的灯笼式辉光效果
- 统计卡片数字动画
- 页面进场动效
- Plus Jakarta Sans / DM Sans / JetBrains Mono 三种非系统字体加载
- Ant Design ConfigProvider darkAlgorithm 全局暗色主题配置
- 空/加载/失败/无结果四种状态

这些效果的实现（相比普通 Ant Design Light 主题）至少需要额外 0.5~1 天的工时，尤其是：
- 字体加载方案配置（fontsource npm 包或 Google Fonts CDN）
- Ant Design 暗色主题 Token 全局覆盖
- 毛玻璃效果与表格组件兼容性调试
- 自定义渐变色装饰线与 Ant Design Table 行样式整合

**建议修复**:

将 Phase 3 总工时从 1.5 天调整为 **2~2.5 天**，或在说明中标注暗色主题效果为"锦上添花"（nice-to-have），核心功能先行实现。

### W3. 缺少前端 CSS 变量/主题配置文件的任务

**文件位置**: 第 5 节（Phase 3）  
**严重性**: [W] Warning  
**问题描述**:

**建议修复**:

在 Phase 3 中新增一个子任务（建议放在 P3A 之后、P3C 之前）：
- `P3A.5 主题配置`：创建 `src/theme/index.ts`，定义 CSS 设计令牌变量和 Ant Design ConfigProvider 主题配置。

# Agent Learning - 智能体持续学习沉淀

## Description

此技能用于记录项目中的关键决策、技术选型、最佳实践、踩坑经验，形成可复用的知识库，帮助智能体持续学习和成长。

## Usage Scenario

- 做出重要技术决策后
- 解决复杂问题后
- 完成重要功能开发后
- 遇到和解决Bug后
- 项目里程碑节点
- 技术选型对比分析

## Instructions

### 1. 学习记录格式

在项目根目录创建 `LEARNINGS.md` 或 `.trae/learnings/` 目录存放学习记录。

#### 基本记录结构

```markdown
## [日期] - [主题]

### 场景描述
[简要描述问题/场景]

### 决策/解决方案
[详细描述做出的决策或解决方案]

### 原因分析
[解释为什么选择这个方案]

### 经验教训
[从中学到的经验]

### 相关文件
- [文件路径]
- [文件路径]
```

### 2. 决策记录模板

当面临技术选型时，使用此模板：

```markdown
## [日期] - [技术选型主题]

### 选项对比

| 选项 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 选项A | ... | ... | ... |
| 选项B | ... | ... | ... |

### 最终选择
[选择的选项]

### 决策理由
1. ...
2. ...
3. ...

### 风险评估
- 潜在风险1: ...
- 缓解措施: ...
```

### 3. 问题解决记录模板

当解决复杂问题后，使用此模板：

```markdown
## [日期] - [问题描述]

### 问题现象
[描述问题的表现]

### 排查过程
1. [第一步尝试]
2. [第二步尝试]
3. ...

### 根本原因
[分析问题的根本原因]

### 解决方案
[描述最终的解决方案]

### 预防措施
[如何避免类似问题再次发生]

### 相关代码/文件
- [文件路径]
```

### 4. 最佳实践记录模板

```markdown
## [日期] - [最佳实践主题]

### 实践描述
[描述这个最佳实践]

### 为什么重要
[解释为什么这是最佳实践]

### 代码示例
[提供代码示例]

### 适用场景
[描述适用场景]
```

### 5. 学习记录分类建议

按以下分类存放学习记录：

- `technical-decisions/` - 技术决策
- `bug-resolutions/` - Bug解决
- `best-practices/` - 最佳实践
- `architecture/` - 架构设计
- `performance/` - 性能优化
- `security/` - 安全相关

## Examples

### 示例1: 数据库选型记录

```markdown
## 2026-04-29 - 数据库选型决策

### 场景描述
项目需要选择关系型数据库，候选方案：PostgreSQL、MySQL、SQL Server

### 选项对比

| 选项 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| PostgreSQL | 功能强大、开源、支持复杂查询 | 学习曲线稍陡 | 复杂业务场景 |
| MySQL | 简单易用、生态成熟 | 某些高级功能有限 | 标准Web应用 |
| SQL Server | 与Windows生态集成好 | 商业授权费用高 | 企业级Windows环境 |

### 最终选择
PostgreSQL

### 决策理由
1. 项目需要复杂查询和JSON支持
2. 开源免费，成本可控
3. 团队有PostgreSQL经验
4. 社区活跃，文档丰富

### 风险评估
- 潜在风险: 团队成员需要学习新特性
- 缓解措施: 提供培训文档和代码示例
```

### 示例2: Bug解决记录

```markdown
## 2026-04-29 - 深色模式切换Bug解决

### 问题现象
切换深色模式时，某些组件样式没有正确更新

### 排查过程
1. 检查CSS变量是否正确定义
2. 检查组件是否正确使用CSS变量
3. 发现某些组件使用了硬编码颜色

### 根本原因
组件中存在硬编码的颜色值，没有使用CSS变量

### 解决方案
将所有硬编码颜色替换为CSS变量：
- `#666` → `var(--text-secondary)`
- `#999` → `var(--text-muted)`
- `#f5f5f5` → `var(--bg-primary)`

### 预防措施
1. 在代码审查中检查硬编码颜色
2. 提供CSS变量使用指南
3. 使用ESLint规则检测硬编码颜色

### 相关代码/文件
- `frontend/src/index.css` - CSS变量定义
- `frontend/src/pages/SqlEditorPage.tsx` - 修复的组件
- `frontend/src/components/Layout.tsx` - 修复的组件
```
